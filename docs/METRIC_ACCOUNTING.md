## Why the video says 22 and the final export says 21

The published video at <https://youtu.be/reSHijMlEnU> preserves the earlier
22-link narration while the final overview/export reports 21/21. The difference
comes from a stricter final provenance decision:

- An earlier review state counted a derived link between a public-price API
  candidate and an XLSX item.
- Final provenance review determined that the candidate was not an exact
  document-item match.
- The derived link was removed; the API record remains available only through
  its separate public-API provenance.

The final packet records `document_evidence_id: null` and
`match_status: "PUBLIC API only · No exact document item match in the attachment
package"`. The aggregation method also says that no exact document-item match is
asserted. The binding manifest therefore contains no API-to-XLSX Evidence link.

Therefore **22 is the pre-final provenance count**, while 21 is the stricter
validated submission metric. The reduction is intentional: RATIN discarded a
weak derived link rather than preserving an unsupported relationship. Devpost
copy, README claims, tests, and manifests must use 21/21 logical links, 26 exact
targets, and zero fallback.
